# コード品質レビュー

## 対象の判定基準

`.py` ファイルまたは `templates/` 配下の `.html` ファイルに変更がある場合

## チェック観点

バグとセキュリティの両面から差分をレビューする。

### バグ
- エッジケースの見落とし
- None/null の未処理
- 非同期処理の問題

### テンプレートの表示ロジック
- ループやフィルタで表示するデータが正しくスコープされているか（例: 全ログから現在のフェーズに関係ないデータまで表示していないか）
- 死亡プレイヤーや過去フェーズのデータが意図せず表示に含まれていないか

### セキュリティ
- ハードコードされた秘密情報
- インジェクション脆弱性（SQL、XSS、コマンド等）
- ユーザー入力のバリデーション不足
- `.env` がコミットに含まれていないか

### 対象外（自動チェック済み）
- 型の不整合 → mypy でカバー
